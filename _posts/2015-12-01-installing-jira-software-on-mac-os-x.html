Brief background

There are some helpful articles on the net on how to setup JIRA Server on Mac. But for a novice Mac OS X user, the instructions may seem a little difficult to follow. Hence this blog post - for the not too terminal-savvy's to help speed up the setup process.

NOTE: Many of the instructions are directly taken from Atlassian's official documentation, referred to hereafter as "the document". So, it may be helpful to have that article too alongside, for quick reference or clarification.

 

1. Setting up Java on Mac

1.1. Installing Java

You need to make sure you have Java 7 or above in your Mac. See JIRA supported platforms for more info. If you already have that in place, you can skip to Step 1.2.

Here is Java's official download page. You can download either the JDK or the JRE software for "Mac OS X x64", but based on my experience I would highly recommend you to install JDK to have anxiety-free JIRA setup.

1.2 Make sure Java is good to use

Atlassian asks you to ensure JAVA_HOME variable is set correctly. If you installed Java SDK as suggested in 1.1 you need not do anything further and can go straight to Step 2. The SDK installer takes care of it, and such that you don't even need the environment variable JAVA_HOME.

Else, run the following commands to check Java version and the variable name. Again, please refer to the official doc for more details.

  1. Open your terminal and type (without the "$", of course): 
    $ java -version
    1. If you see an output saying something like this, you are good: 
      java version "1.8.0_66"
Java(TM) SE Runtime Environment (build 1.8.0_66-b17)
Java HotSpot(TM) 64-Bit Server VM (build 25.66-b17, mixed mode)
    2. If you see an alert like this, "to use the 'java' command-line tool you need to install a jdk", then please do go back to Step 1.1. That's going to be the easiest way out for you.
  2. Just as a prep to the next step, this command prints the value of JAVA_HOME. 
    $ echo $JAVA_HOME

1.3. Setup JAVA_HOME Environment Variable

To explicitly set JAVA_HOME environment variable, you may have to create or update a hidden file, ~/.bash_profile. Following are the commands. (Reference: Stack Overflow)

$ echo export "JAVA_HOME=\$(/usr/libexec/java_home)" >> ~/.bash_profile
$ source .bash_profile
$ echo $JAVA_HOME
$ #This should output a Java directory. You're all set.

 

2. Download and Install JIRA

  1. Download the JIRA tar.gz file here.
  2. Unzip the downloaded file.
  3. Move it to another location if you want. I would suggest: /Users/your_user_name/.
  4. This location is your "JIRA Installation Directory".

 

3. Create a Dedicated User Account to Run JIRA

At this point, the document asks you to "Set JIRA Home". But, we are going to do that in the next step. We'll create a user account first. The shell commands unfamiliar to me, so I just created it from UI.

  1. Go to System Preferences > Users and Groups
  2. Click on the Lock symbol at bottom-left to make changes. (You'll be asked for your current user password. Enter that.)
  3. Click on the [+] icon right above the lock to create a new account.
    1. Set New Account to "Sharing Only".
    2. Add Full Name and Account Name as "jira".
    3. "Use separate password" and set that up.
    4. "Create User".

 

4. Set JIRA Home

You can specify any location on a disk for your JIRA Home Directory, however please read the document's notes on which locations to avoid and why.

I would suggest create your JIRA Home Directory in here: /Users/jira/atlassian/jira-home, because else I ran into folder permissions issues when running the website later on.

You will also be required to specify the location of your JIRA Home Directory in to JIRA's configuration files and set an environment variable:

  1. Go to <JIRA Installation Directory>/atlassian-jira/WEB-INF/classes/jira-application.properties/jira-application.properties file and open.
  2. Set the value of the 'jira.home' property to your JIRA Home Directory. 
    jira.home = /Users/jira/atlassian/jira-home
  3. Set an environment variable named JIRA_HOME = <JIRA Home Directory>. Following are the commands: 
    $ echo export JIRA_HOME=/Users/jira/atlassian/jira-home >> ~/.profile
$ source ~/.profile
$ echo $JIRA_HOME
$ #This should print JIRA_HOME value that you just set

 

5. Set Folder Permissions

As the final steps, you need to set permissions to jira user on the following folders:

<JIRA Installation Directory>/logs
<JIRA Installation Directory>/work
<JIRA Installation Directory>/temp
<JIRA Home Directory>

How to set Permissions from UI

Go to the folder directory and "Get Info". The bottom-most section defines user permissions for that folder.
Add new entry to give jira read/write permissions.
Mark "jira" the owner.
Make sure you "apply enclosed items" too.

7. It's Show Time

You're all set now. In your terminal, go to <JIRA Installation Directory>/bin. Switch user to jira and run the application startup script:

$ sudo -s -u jira
$ Password: [] #enter your current account password
$ ./startup.sh

Your JIRA server (on TomCat web server) should start-up. (Woo-hoo!) Go ahead and hit your browser with: http://localhost:8080/ and see what you get.

Other stuffs

Some other notes that might be handy:

  • It is a good idea to check your TomCat server logs to confirm that everything went smoothly. It should be located in <JIRA Installation Directory>/logs/catalina.out folder.
  • If your port 8080 is already taken, change your JIRA config before running the startup script (more info).
  • If you want to shutdown TomCat server, run the shutdown.sh script.
